## Typography

**Font Stack:**
```css
font-family: 'Noto Sans TC', 'PingFang TC', 'Microsoft JhengHei', '微軟正黑體', Arial, sans-serif;
```

Noto Sans TC is loaded from Google Fonts (weights 400, 500, 700) and must remain in `app/layout.tsx`'s `<head>`. Do not remove the `preconnect` links.

**Scale:**

| Class | Size | Usage |
|-------|------|-------|
| `text-6xl font-bold` | 60px | Game board large center text |
| `text-3xl font-bold` | 30px | Page titles (大腹翁) |
| `text-xl font-bold` | 20px | Section headings |
| `text-lg font-bold` | 18px | Button labels (large) |
| `text-base` | 16px | Body text |
| `text-sm font-medium` | 14px | Card labels, player names |
| `text-xs` | 12px | Badges, helper text, captions |

Minimum body size is 16px. Badges and captions are acceptable at 12px (`text-xs`).

## Spacing

Uses the Tailwind default 4px base scale. Common patterns:

| Pattern | Usage |
|---------|-------|
| `p-4` / `p-8` | Card inner padding (small / large) |
| `gap-2` / `gap-3` | List / flex item gaps |
| `mb-4` / `mb-6` | Section separation |
| `px-4 py-3` | Input fields |
| `px-4 py-2` | Standard buttons |
| `py-3` | Large / full-width buttons |

## Border Radius

Three tiers — do not mix:

| Class | Usage |
|-------|-------|
| `rounded-full` | Player avatars, badges/pills, progress bar fills |
| `rounded-xl` | Buttons, input fields, inner cards, board squares |
| `rounded-2xl` | Major containers (modal cards, side panels) |

Inner elements nested inside `rounded-2xl` containers should use `rounded-xl`, never `rounded-2xl` again (inner ≤ outer).

## Background

The page background is a warm cream with a subtle dot grid that evokes board-game graph paper:

```css
background-color: var(--bg-page);          /* #fef9ef */
background-image: radial-gradient(circle, rgba(180, 120, 30, 0.22) 1.5px, transparent 1.5px);
background-size: 24px 24px;
```

Do not change the dot color or size — it is intentionally subtle. The dots should not be visible as a dominant pattern, only as a faint texture.

## Interactive States

All interactive elements must have all applicable states:

| State | Pattern |
|-------|---------|
| Hover (button) | Darken one Tailwind shade: `amber-500 → amber-600` |
| Hover (board square) | `filter: brightness(0.95)` (0.1s transition) via `.board-square:hover` |
| Hover (link/text) | `hover:text-amber-700` or `hover:text-amber-800` |
| Focus (input) | `focus:outline-none focus:border-amber-400` — no bare `outline: none` without replacement |
| Disabled | `disabled:opacity-40` — never remove, never hide |
| Current turn | `border-amber-400 bg-amber-50` (PlayerPanel) |
| Bankrupt | `opacity-40` on the entire player row |

Touch targets: minimum **44×44px** on all interactive elements (enforced via `min-w-[44px] min-h-[44px]` where needed).

## Motion & Animation

Two defined animations in `app/globals.css`:

**Dice roll** (`.dice-rolling`):
- Duration: `0.5s ease-in-out infinite`
- Keyframes: rotate ±15° + scale 0.9–1.2
- Only `transform` is animated (no layout properties)

**Confetti fall** (`.confetti-piece`):
- Duration: 2–4s per piece (randomised in JS), `linear forwards`
- Keyframes: `translateY(-20px → 100vh)` + `rotate(720deg)` + `opacity: 1 → 0`

**Token movement:**
- `transition-all duration-500` on board position

**General transitions:**
- `transition` (150ms default) on buttons and borders
- Never use `transition: all` — prefer listing specific properties

**Accessibility:** Both `.dice-rolling` and `.confetti-piece` are disabled under `@media (prefers-reduced-motion: reduce)`. Any new animation must add a corresponding rule to this block.

## Progress Bar

Children progress toward `WIN_CHILDREN` target:

```jsx
<div className="bg-gray-200 rounded-full h-1.5">
  <div className="bg-pink-400 rounded-full h-1.5 transition-all" style={{ width: `${percent}%` }} />
</div>
```

Color is `bg-pink-400` (thematic — babies/romance). Do not change to amber.

## Layouts

**Page shell:** `min-h-screen flex items-center justify-center p-4`

**Game layout:** `lg:flex-row` split — board (left, `flex-shrink-0`) + sidebar (right, `flex-1 min-w-0`)

**Responsive breakpoints used:** `md:` (768px), `lg:` (1024px)

Mobile: single column, sidebar panels collapse below board.

## AI Slop — What to Avoid

This project has a strong thematic identity. Do not introduce:

- Purple/indigo gradients or blue-to-purple color schemes
- 3-column feature grids (icon + title + description × 3)
- Icons in colored circles as decoration
- Centered text on all headings
- Uniform `rounded-2xl` on everything (see border radius tier system above)
- Decorative blobs or wavy dividers
- Generic hero copy

The emoji usage (🤰, 🎲, 👶, 💰, 🍼) is **intentional brand** — not AI slop. The mascot emoji is the logo. New screens may introduce new thematic emoji but should use them sparingly and purposefully.
